# UX Philosophy — App BiT

## Principio Central

App BiT no es una plataforma tecnológica.

App BiT es un instrumento de comprensión territorial.

Su propósito no es mostrar datos.
Su propósito es ayudar a comprender realidades humanas que normalmente quedan ocultas detrás de estadísticas dispersas.

Cada pantalla debe transmitir:

* Confianza institucional
* Inteligencia analítica
* Profundidad investigativa
* Impacto social
* Claridad en la toma de decisiones

El usuario debe sentir que está consultando una fuente estratégica de información, no navegando un software comercial.

---

# Personalidad Visual

Si App BiT fuera un espacio físico sería:

* Una sala de análisis territorial
* Un centro de investigación pública
* Una mesa de trabajo llena de mapas, informes y evidencia
* Un laboratorio de políticas públicas

Nunca debe parecer:

* Una startup de IA
* Una fintech
* Un CRM
* Un dashboard corporativo tradicional
* Un SaaS genérico

---

# Lenguaje Emocional

La experiencia debe generar:

1. Curiosidad

"¿Qué está ocurriendo realmente en esta región?"

2. Comprensión

"Ahora entiendo el problema."

3. Descubrimiento

"No había visto esta relación entre indicadores."

4. Acción

"Ya sé dónde intervenir."

Todo el recorrido debe llevar al usuario desde la observación hasta la decisión.

---

# Atmósfera Visual

La interfaz debe sentirse como una mezcla entre:

* Cartografía moderna
* Sistemas GIS
* Inteligencia territorial
* Investigación académica
* Infraestructura digital

El contraste entre Butter e Iris representa el corazón conceptual del producto:

Butter:

* Conocimiento
* Documentación
* Evidencia
* Archivos
* Investigación

Iris:

* Profundidad
* Inteligencia
* Confianza
* Estrategia
* Decisión

La unión de ambos comunica:

"Información transformada en criterio."

---

# Jerarquía de Información

La información es protagonista.

Las decoraciones nunca deben competir con los datos.

Reglas:

* El contenido siempre domina sobre los efectos visuales.
* El usuario debe encontrar contexto antes que métricas.
* Las visualizaciones deben explicar, no impresionar.
* Los números deben contar una historia.

---

# Diseño del Espacio

Evitar layouts excesivamente centrados.

Priorizar:

* Alineaciones a la izquierda
* Lectura tipo informe ejecutivo
* Composición editorial
* Grillas amplias
* Ritmo vertical consistente

Inspiración:

* Financial Times
* Bloomberg
* ArcGIS
* Palantir
* Notion Intelligence
* Atlas de datos territoriales

---

# Movimiento y Animación

Las animaciones deben comunicar flujo de información.

Nunca entretenimiento.

Permitir:

* Apariciones suaves
* Transiciones contextuales
* Resaltado de relaciones
* Flujo entre nodos
* Énfasis en cambios de estado

Evitar:

* Rebotes
* Efectos exagerados
* Animaciones largas
* Elementos distractores

Regla:

Si una animación no mejora la comprensión, debe eliminarse.

---

# Uso del Color

No utilizar blanco puro.

No utilizar negro puro.

La interfaz debe sentirse cálida, documental y profesional.

Reglas:

* Butter para superficies de lectura.
* Iris para estructura y jerarquía.
* Naranja únicamente para acciones e insights importantes.
* Heatmaps respetando siempre la escala oficial.
* Los colores deben informar antes que decorar.

---

# Principio de Diseño Final

Cada pantalla debe responder una única pregunta:

"¿Estoy ayudando al gestor público a entender mejor una realidad que antes no podía ver?"

Si la respuesta es no, el diseño debe simplificarse.
